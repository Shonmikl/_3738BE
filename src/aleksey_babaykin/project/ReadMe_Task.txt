Задача (первоначальное описание):
Создать систему по хранению электронных файлов на, возможно ,удаленном ресурсе, при этом хранение должно поддерживать:
1. хранение информации о файле
1.1. должно давать возможность хранить поля:
1.1.1 поля обязательные ( дата создания, тип шаблона описания параметров, ИД файла, информацию о версии, актуальность
    версии)
1.1.2 поля не обязательные, т.е. поля не определенные на этапе разработке, а мользователь должен сам иметь возможность
     такие поля создать в зависимости от потребностей, пример: а) анименование файла б) код клиета в) наименование клиента
     г) пользователя, который сосздал д) системы, которая создала и т.д.
     Ограничения: добавлемые поля может быть одним из типов: а) Строку б) Число ( инт)  в) Дату г) Логическое значение
1.2 версионность:
1.2.1 - механизм определения версии, т.е. если файл с параметрами ( определенными, как обязательные) уже имеется в системе,
      то файл с такими параметрами определяется, как файл с устаревшей версией, а новоу присваевается тоже ИД версии, но
      при этом проставляется параметр, что файл последний в цепочке версионности.
      ( пример полей: наименование клиента + тип документа)
1.2.2 - если параметры не установлены или по тем параметрам, что установлены , нет размещенного файла, то документ размещается,
       как первый в своей цепочке.

2. Размещение в структуру папок,
2.1. - физическое размещение: происходит в зараенее созданные Х папок, при этом с +-нормальным стахостическим
       распределением.
2.2. - размещение описательной информации происходит в структуру, покожую на папки.
2.3. - файл в папке размещается исключительно для визуализации.
2.4. - одна и таже информация о файле визуально может быть привязана к нескольким папкам.
2.5 - если файл не привязан к папке, он попадает в общую папку и может там хранится ( хотя это будет как логическая ,
      но не программная ошибка)
3.1 - Должно быть реализован АПИ, которое позволит стороннему приложению по https взаимодействовать с указанным решением
4.1. Для АПИ реализовать секьюритизацию;
     Ограничание прав доступа для Папаок и Файлов будут реализованы после разработки п. 1. - 3.
5.1. Реализация Веб Интерфейса планируется сделать после реализации 1. - 3.

Примечание:
- не все возможности будут реализованы сразу
- файл может быть дополнен по мере разработки.
